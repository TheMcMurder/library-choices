## Interaction Contract

### Slider Behavior

| Interaction | Expected Behavior |
|-------------|-------------------|
| Drag (mouse/touch) | `input` event fires continuously; `#collections-amount`, `#collections-description`, `aria-valuetext`, and `#result-amount` all update mid-drag |
| Arrow key | Each key press moves one $10k step; same live update as drag |
| Home / End key | Jumps to $10,000 / $60,000; same live update |
| Page Up / Down | Browser default (large step); same live update |
| Release / blur | `change` event fires; URL updated via `history.replaceState` (existing url.js behavior) |
| Page load (no URL param) | Slider at default node ($30,000); description and `aria-valuetext` initialized from `LIBRARY_DATA` |
| Page load (`?collections=30000`) | Slider restored to $30,000 node; `aria-valuetext` set to match restored value before first user interaction |

### `aria-valuetext` Format

Every slider position must announce: `"30,000 dollars — Print collection + digital"`

Pattern: `{value/1000}.toLocaleString('en-US'),000 dollars — {node.description}`

The em dash (`—`) is the Unicode character U+2014. Do not use a hyphen.

### Slider CSS — Required Rules

Apply in `@layer base` in `src/css/style.css`:

```
input[type='range'] — -webkit-appearance: none; appearance: none; background: transparent; cursor: pointer; width: 100%
::-webkit-slider-runnable-track — background: blue-200; border-radius: 0.5rem; height: 0.5rem (8px)
::-webkit-slider-thumb — background: blue-800; border-radius: 50%; height: 1.25rem (20px); width: 1.25rem; margin-top: -0.375rem (-6px)
::-moz-range-track — background: blue-200; border-radius: 0.5rem; height: 0.5rem
::-moz-range-thumb — background: blue-800; border: none; border-radius: 50%; height: 1.25rem; width: 1.25rem
:focus-visible — outline: 2px solid blue-600; outline-offset: 2px
```

Thumb dimensions: 20px diameter. Track height: 8px. Vertical alignment formula: `margin-top = -(20px - 8px) / 2 = -6px`.

Source: `07-RESEARCH.md` Pattern 5.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| SLDR-05: meaningful screen reader label | `aria-valuetext` set at page load and on every `input` event — never stale |
| SLDR-01: keyboard navigation | Native `<input type="range">` provides arrow keys, Home/End, Page Up/Down for free |
| TRST-03: 44px touch targets | Slider thumb is 20px; wrap entire slider control area with sufficient vertical padding to meet 44px total interactive height (existing `min-h-[44px]` pattern) |
| Focus visible ring | `focus-visible` CSS on range input with `outline: 2px solid blue-600; outline-offset: 2px` — consistent with existing radio/checkbox pattern |
| Label row hidden from AT | `aria-hidden="true"` on the tick label `<div>` — screen reader gets value from `aria-valuetext` |
