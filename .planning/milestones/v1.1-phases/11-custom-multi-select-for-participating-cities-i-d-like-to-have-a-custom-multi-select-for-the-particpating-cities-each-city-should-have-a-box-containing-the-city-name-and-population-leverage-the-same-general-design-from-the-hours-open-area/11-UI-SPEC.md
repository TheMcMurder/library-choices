## Design Pattern Reference: Hours Open Cards (Phase 10)

The requirement explicitly states to "leverage the same general design from the hours open area." The following card pattern from Phase 10 is the direct visual reference and must be matched exactly.

**Existing Hours Open card markup pattern:**

```html
<label
  for="staffing-{id}"
  class="block cursor-pointer rounded-lg border p-6 transition-colors
         has-[:checked]:bg-blue-50 has-[:checked]:ring-2 has-[:checked]:ring-blue-600
         bg-gray-50 border-gray-200
         has-[:focus-visible]:ring-2 has-[:focus-visible]:ring-blue-600
         min-h-[44px]"
>
  <input type="radio" class="sr-only" ... />
  <span class="text-base font-semibold text-gray-900">...</span>
</label>
```

**City card adaptation (checkbox, not radio):** Replace `type="radio"` with `type="checkbox"`. The `has-[:checked]` CSS selector works identically for checkboxes. All other classes carry over unchanged.

## Component Anatomy: City Card

Each city card contains:

1. **Hidden checkbox input** — `class="sr-only"`, `type="checkbox"`, `name="cities"`, `value="{city.id}"`, `data-households="{city.households}"`
2. **City name line** — `text-base font-semibold text-gray-900`
3. **Population line** — `text-sm text-gray-500` — displays household count formatted with commas (e.g., "2,823 households")
4. **Source cite** — `text-sm text-gray-500 not-italic` — links to census source via `<a>` inside `<cite>`

**Card wrapper element:** `<label>` wrapping the hidden checkbox, identical to the staffing card pattern.

**Fieldset wrapper:** `<fieldset class="bg-gray-50 border border-gray-200 rounded-lg p-6 space-y-4">` with `<legend class="text-xl font-semibold text-gray-900 px-1">Participating Cities</legend>`

## Spacing Scale

Declared values (multiples of 4). All values match Tailwind's default spacing scale already used in the project.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Gap between city name and population within a card (`mt-1` / `gap-2`) |
| md | 16px | Default element spacing |
| lg | 24px | Card internal padding (`p-6` = 24px) |
| xl | 32px | Layout gaps between fieldsets (`space-y-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Minimum touch target: `min-h-[44px]` on each city card wrapper (matches TRST-03 requirement; matches Phase 10 staffing cards)

**Source:** `src/index.html` existing `p-6`, `space-y-8`, `min-h-[44px]` usage

## Interaction Contract

### Selected State (CSS-only, no JavaScript)

Selection state is driven entirely by `has-[:checked]` CSS selector — no JavaScript class toggling.

| State | Classes applied to card wrapper |
|-------|--------------------------------|
| Unselected | `bg-gray-50 border-gray-200` |
| Selected | `has-[:checked]:bg-blue-50 has-[:checked]:ring-2 has-[:checked]:ring-blue-600` |
| Focus-visible | `has-[:focus-visible]:ring-2 has-[:focus-visible]:ring-blue-600` |

**Note:** All four cities start checked (`defaultChecked: true` in config.js). The card must render with full checked appearance on initial page load.

### Keyboard Navigation

- `Tab` moves focus into and between city cards
- `Space` toggles the checkbox inside the focused card
- Focus ring appears on the card wrapper (not the hidden checkbox) via `has-[:focus-visible]`
- Arrow keys are NOT applicable — checkboxes do not follow radio group arrow-key behavior

### Zero-City Guard

The existing CONF-05 zero-city guard in `calculator.js` must continue to work. No changes to guard behavior. When all cities are unchecked, the result bar shows the friendly message (already implemented). The card UI itself has no error state — the guard is handled downstream.

### URL Encoding

City selections are encoded via `phi` compact param (comma-joined positional indices) by `url.js`. This phase does not modify `url.js`. The new checkbox inputs must preserve the existing `name="cities"` attribute and `value="{city.id}"` format so the existing form delegation in `calculator.js` continues to work without changes.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Section heading | "Participating Cities" (unchanged from current) |
| Card city name | `{city.label}` — e.g., "Providence" |
| Card population line | `{city.households | toLocaleString} households` — e.g., "2,823 households" |
| Source citation | `Source: {city.source}` — links to `{city.sourceLink}` if present |
| Empty state heading | Not applicable — zero-city guard message is already implemented in `calculator.js` / result bar, not in this fieldset |
| Error state | Not applicable — no validation errors in city selection UI |
| Destructive confirmation | Not applicable — no destructive actions |

**Population label note:** The config stores `households` (not raw population). The display label reads "households" to remain accurate. Changing to "population" would be inaccurate.

**Source of copy convention:** `src/index.html` existing city fieldset, `src/_data/config.js` city objects

## Component Inventory

### New: City Card (`<label>` wrapping `<input type="checkbox">`)

Replaces the current `<div class="flex items-center gap-2">` + separate `<label>` + `<cite>` pattern with a single full-width card, matching the Phase 10 staffing card pattern.

**Before (current):**
```html
<div class="flex flex-col gap-2">
  <div class="flex items-center gap-2 min-h-[44px]">
    <input type="checkbox" name="cities" ... class="accent-blue-600 ..." />
    <label for="city-{id}" class="text-base font-semibold text-gray-900 cursor-pointer">
      {city.label} <span class="text-sm text-gray-500">({households} households)</span>
    </label>
  </div>
  <cite class="text-sm text-gray-500 not-italic">Source: {city.source}</cite>
</div>
```

**After (Phase 11 target):**
```html
<label
  for="city-{city.id}"
  class="block cursor-pointer rounded-lg border p-6 transition-colors
         has-[:checked]:bg-blue-50 has-[:checked]:ring-2 has-[:checked]:ring-blue-600
         bg-gray-50 border-gray-200
         has-[:focus-visible]:ring-2 has-[:focus-visible]:ring-blue-600
         min-h-[44px]"
>
  <input
    type="checkbox"
    name="cities"
    id="city-{city.id}"
    value="{city.id}"
    data-households="{city.households}"
    class="sr-only"
    {%- if city.defaultChecked %}checked{% endif %}
  />
  <span class="text-base font-semibold text-gray-900">{{ city.label }}</span>
  <p class="text-sm text-gray-500 mt-1">{{ city.households | toLocaleString }} households</p>
  <cite class="text-sm text-gray-500 not-italic mt-1 block">Source: {{ city.source }}</cite>
</label>
```

### Unchanged: Fieldset wrapper

```html
<fieldset class="bg-gray-50 border border-gray-200 rounded-lg p-6 space-y-4">
  <legend class="text-xl font-semibold text-gray-900 px-1">Participating Cities</legend>
  {# city cards loop here #}
</fieldset>
```

## Data Contract

No new data fields required. All city rendering is driven by the existing `config.cities` array shape:

| Field | Type | Display role |
|-------|------|-------------|
| `id` | string | `for`, `id`, `value` attributes |
| `label` | string | City name (heading line of card) |
| `households` | number | Population display (formatted with `toLocaleString`) |
| `defaultChecked` | boolean | Initial `checked` attribute |
| `source` | string | Citation text |
| `sourceLink` | string (optional) | Citation href (if present, wrap source in `<a>`) |
