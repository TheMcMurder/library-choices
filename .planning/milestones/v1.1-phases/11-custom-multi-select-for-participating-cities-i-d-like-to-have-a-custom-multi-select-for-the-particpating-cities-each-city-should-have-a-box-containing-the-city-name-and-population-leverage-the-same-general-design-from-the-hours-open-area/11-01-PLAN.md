Critical integration contract preservations:
- `name="cities"` — calculator.js queries `input[name="cities"]:checked`
- `value="{{ city.id }}"` — url.js reads `cb.value` for phi param encoding
- `data-households="{{ city.households }}"` — calculator.js reads `cb.dataset.households` in getTotalHouseholds()
- `{% if city.defaultChecked %}checked{% endif %}` — bare attribute pattern, NOT `checked="{{ city.defaultChecked }}"`
- `type="checkbox"` — NOT radio; checkboxes allow independent multi-select

Anti-patterns to avoid:
- Do NOT use `accent-blue-600` on the checkbox — it is `sr-only` (visually hidden), so accent has no effect
- Do NOT add `checked="{{ city.defaultChecked }}"` — HTML treats `checked="false"` as checked
- Do NOT add margin between cards — the fieldset `space-y-4` class handles card-to-card gap
- Do NOT add any JavaScript for selection state — `has-[:checked]` CSS handles it
- Do NOT modify calculator.js or url.js — the card markup preserves all existing integration contracts
