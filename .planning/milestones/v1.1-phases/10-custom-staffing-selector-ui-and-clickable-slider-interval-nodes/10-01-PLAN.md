Two additions to `src/js/calculator.js`, both inside the existing IIFE + `'use strict'` wrapper.

**Addition 1 — Extend `updateSliderLabels()` (per D-14, D-16):**

After the existing line `slider.setAttribute('aria-valuetext', ...)` (currently line 57), add this block:

```javascript
    var nodeButtons = document.querySelectorAll('[data-value]');
    nodeButtons.forEach(function (btn) {
      var isActive = btn.dataset.value === String(slider.value);
      btn.classList.toggle('text-blue-800', isActive);
      btn.classList.toggle('font-semibold', isActive);
      btn.classList.toggle('text-gray-500', !isActive);
      btn.classList.toggle('font-normal', !isActive);
    });
```

This toggles active style (`text-blue-800 font-semibold`) on the matching button and inactive style (`text-gray-500 font-normal`) on all others. Uses `String(slider.value)` for safe string comparison with `btn.dataset.value`.

Note: The `[data-value]` selector is scoped by the fact that only the slider tick buttons have `data-value` attributes in the current markup. Per Research Pitfall 3, this is acceptable for now — if future phases add `data-value` elsewhere, the selector should be scoped to the tick container.

**Addition 2 — Register node button click handlers (per D-12, D-17):**

Add this block in the initialization section at the bottom of the IIFE, AFTER the existing `updateResult()` and `updateSliderLabels()` calls (currently lines 88-89) and BEFORE the closing `}());`:

```javascript
  // Node button click handlers — snap slider to clicked value
  document.querySelectorAll('[data-value]').forEach(function (btn) {
    btn.addEventListener('click', function () {
      var slider = document.getElementById('collections');
      slider.value = btn.dataset.value;
      slider.dispatchEvent(new Event('input', { bubbles: true }));
      slider.dispatchEvent(new Event('change', { bubbles: true }));
    });
  });
```

Per D-17: Events dispatch on the `#collections` slider element with `{ bubbles: true }` so they reach the existing `form.addEventListener('input', ...)` and `form.addEventListener('change', ...)` delegation. Event order: `input` then `change` — `input` fires `updateSliderLabels()` (which syncs node active state), `change` fires `updateResult()` + `encodeUrl()`.

**Per D-18:** No changes to url.js. The `tau` encoding reads `slider.value` directly; clicking a node sets that value before events fire.

Do NOT add any new functions. Do NOT modify `getStaffingCost()`, `getCollectionsCost()`, `getTotalHouseholds()`, `updateResult()`, or any event listener registrations. Only extend `updateSliderLabels()` and add the click handler registration block.
