**eleventy.config.js — Register formatDays filter:**

Add the `formatDays` filter immediately after the existing `toLocaleString` filter (before the `return` statement). Exact implementation:

```javascript
eleventyConfig.addFilter("formatDays", (daysStr, locale) => {
  locale = locale || "en-US";
  const REFERENCE_MONDAY = new Date("2024-01-01T12:00:00Z");
  const fmt = new Intl.DateTimeFormat(locale, { weekday: "long", timeZone: "UTC" });
  const parts = daysStr.split("-").map(Number);
  if (parts.length === 1) {
    const d = new Date(REFERENCE_MONDAY);
    d.setUTCDate(d.getUTCDate() + (parts[0] - 1));
    return fmt.format(d);
  }
  const startDate = new Date(REFERENCE_MONDAY);
  startDate.setUTCDate(startDate.getUTCDate() + (parts[0] - 1));
  const endDate = new Date(REFERENCE_MONDAY);
  endDate.setUTCDate(endDate.getUTCDate() + (parts[1] - 1));
  return fmt.format(startDate) + "\u2013" + fmt.format(endDate);
});
```

Key details:
- `2024-01-01T12:00:00Z` — noon UTC prevents cross-midnight timezone bugs (per RESEARCH.md Pitfall 2)
- `timeZone: "UTC"` in Intl options — prevents CI/CD timezone drift
- `parts.length === 1` branch handles single days like `"6"` (per Pitfall 1)
- `\u2013` is en dash between day range names (Monday–Friday)
- Default locale `"en-US"` with overridable param per D-08
