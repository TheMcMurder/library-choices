  <action>
    **Create .github/workflows/test.yml (per D-08, D-09, D-10):**
    ```yaml
    name: Tests

    on:
      push:
        branches: [main]
      pull_request:
        branches: [main]

    jobs:
      test:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - name: Setup pnpm
            uses: pnpm/action-setup@v4
          - name: Setup Node
            uses: actions/setup-node@v4
            with:
              node-version: "22"
              cache: "pnpm"
          - run: pnpm install --frozen-lockfile
          - run: pnpm test
    # Non-blocking: no dependency from deploy.yml
    # To promote to blocking gate: add `needs: test` to deploy.yml's build job
    ```

    This workflow:
    - Runs on push to main AND pull requests to main
    - Uses the same pnpm/node setup pattern as deploy.yml
    - Runs `pnpm test` (which runs `vitest run`)
    - Is completely independent from deploy.yml — no `needs` relationship
    - Does NOT modify deploy.yml in any way

    Do NOT touch .github/workflows/deploy.yml.
  </action>
