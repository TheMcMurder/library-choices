## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | white (`bg-white`) | Page background, card interior when unselected |
| Secondary (30%) | gray-50 (`bg-gray-50`) | Card default background, fieldset backgrounds, footer |
| Accent (10%) | blue-800 (`bg-blue-800`) | Header, result bar, selection ring (has-[:checked]:ring-blue-600), slider thumb |
| Current-level indicator | amber-500 (`#f59e0b`) | Current-level badge background, current-level card border, current-level slider tick — this phase only |
| Destructive | red-600 (`text-red-600`) | Draft watermark only — no destructive actions in this phase |

Accent reserved for: selection ring on checked cards, focus outline on interactive elements, header background, result bar background, slider thumb, slider track, "Show cost breakdown" button hover state.

**Current-level amber reserved for:** staffing card amber border ring (`ring-amber-500`), "Current level" badge background (`bg-amber-500` text white), collections slider tick mark colored amber (`text-amber-500` or equivalent inline style).

Coexistence rule (CURR-01): The amber border is rendered as a second ring at a larger offset than the blue selection ring so both are simultaneously visible when the card is selected:
- Amber ring: `ring-2 ring-amber-500 ring-offset-2` — always present when `isCurrentServiceLevel: true`
- Blue selection ring: `has-[:checked]:ring-2 has-[:checked]:ring-blue-600` — inside the amber ring
- Focus outline: `has-[:focus-visible]:outline has-[:focus-visible]:outline-2 has-[:focus-visible]:outline-blue-600 has-[:focus-visible]:outline-offset-4` — outermost, already established in Phase 12

Ring layering order (outermost to innermost): focus outline (offset-4) → amber current-level ring (offset-2) → blue selection ring (no offset) → card border.

**Primary visual anchor:** the amber ring and "Current level" badge on the current-service-level staffing card. This is the focal point the user's eye lands on first when scanning the staffing options.

**Source:** codebase scan `src/index.html`, `src/css/style.css`; amber color new for this phase, defaulting to Tailwind amber-500.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | None — this phase adds indicators only; no CTA |
| Badge label | "Current level" |
| Badge screen reader text | "Current level" (same text — sr-only span or aria-label on badge element) |
| Empty state heading | Not applicable — indicators only appear when `isCurrentServiceLevel: true` is set; config always has one such item |
| Error state | Not applicable — no async operations in this phase |
| Destructive confirmation | None — no destructive actions in this phase |

Badge copy rationale: "Current level" is the exact phrase used throughout REQUIREMENTS.md and the success criteria. It is plain language, avoids jargon, and directly answers the citizen's question "which option is the one we have now?"

Slider tick accessibility: The existing slider tick buttons are `aria-hidden="true"` in the template. The current-level tick gets an additional visual treatment (amber color) but because the tick row is already hidden from screen readers, no additional sr-only text is required on the tick itself. The amber card badge (CURR-05) carries the full accessible announcement.

**Source:** REQUIREMENTS.md CURR-02, CURR-05; `src/index.html` existing aria-hidden pattern on tick row

## Component Inventory

New visual elements introduced in this phase:

| Component | Element type | Tailwind classes | Notes |
|-----------|-------------|-----------------|-------|
| Current-level card ring | Static ring on `<label>` | `ring-2 ring-amber-500 ring-offset-2` | Applied when loop item has `isCurrentServiceLevel: true`; coexists with blue checked ring |
| Current-level badge | `<span>` inside `<label>`, top-right | `absolute top-2 right-2 inline-flex items-center rounded-full bg-amber-500 px-2 py-1 text-xs font-semibold text-white` | Requires `relative` on parent `<label>` |
| Screen reader badge text | Inner `<span>` or badge element text | `sr-only` if visually hidden, or plain text inside badge | "Current level" text |
| Amber slider tick | Existing `<button>` tick element | Add `text-amber-600 font-semibold` when tick value matches current-level option value | Amber-600 for better contrast on white background vs amber-500 |

Label positioning note: The card `<label>` must gain `relative` positioning to allow `absolute` badge placement. Existing label classes do not include `relative` — this must be added conditionally or universally on all staffing labels.

**Source:** REQUIREMENTS.md CURR-01 through CURR-05; `src/index.html` existing card and tick markup

## Interaction States

| State | Staffing card (current level) | Collections tick (current level) |
|-------|------------------------------|----------------------------------|
| Default (unselected) | amber ring visible, badge visible | tick text amber-600 |
| Selected (checked) | amber ring + blue selection ring both visible, badge visible | tick text amber-600 (unchanged) |
| Focus | amber ring + blue outline at offset-4, badge visible | existing focus outline on tick button |
| Selected + Focus | all three rings visible simultaneously | tick amber + focus outline |

No JavaScript interaction is required. All indicator states are CSS-only driven by the Nunjucks template conditionally applying classes based on `isCurrentServiceLevel`.

**Source:** STATE.md accumulated decisions — "has-[:checked] CSS-only selection state for cards", "outline-offset-4 for focus rings"
