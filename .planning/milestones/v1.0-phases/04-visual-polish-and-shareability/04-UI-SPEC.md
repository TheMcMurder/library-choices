> Visual and interaction contract for Phase 4: Visual Polish and Shareability.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Tailwind CSS v4 utility classes only |
| Preset | not applicable |
| Component library | none — native HTML elements only |
| Icon library | Heroicons v2 — single inline SVG (question-mark-circle, 20/solid), no package install |
| Font | system-ui stack (`system-ui, sans-serif`) — no web font download |

**Rationale:** Project is Eleventy v3 + Tailwind CSS v4 + vanilla JS. No React, no shadcn, no bundler. All Phase 4 markup is hand-authored native HTML. The single Heroicons SVG is copied inline from heroicons.com — no npm dependency added. Source: 04-RESEARCH.md Standard Stack.

**Confirmed absent:** No `components.json`, no `tailwind.config.*`, no `postcss.config.*`. The only CSS file is `src/css/style.css` containing `@import "tailwindcss"`. No `@theme` blocks needed for Phase 4 — all values are expressible as Tailwind v4 utility classes.

## Spacing Scale

Extends Phase 2 contract (source: `02-UI-SPEC.md`). All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps; `mt-1` subtitle under header |
| sm | 8px | Compact gaps within sticky bar; toggle button padding |
| md | 16px | Horizontal padding on mobile (`px-4`); sticky bar vertical padding (`py-4`) |
| lg | 24px | Internal fieldset padding (inherited from Phase 2) |
| xl | 32px | Horizontal padding on sm+ screens (`px-8`) |
| 2xl | 48px | — |
| 3xl | 64px | Sticky bar height reference; `pb-16` (64px) bottom padding on `<main>` |

**Exceptions:**
- `padding-bottom` on `<main>`: 64px (`pb-16`) — matches sticky bar height and satisfies RESEARCH.md Pitfall 1 guidance. Source: 04-RESEARCH.md Pitfall 1.
- Sticky bar internal padding: `px-4 py-4` (16px horizontal, 16px vertical).
- Header bar padding: `px-4 py-4 sm:px-8` — matches `<main>` breakpoint guard.
- `<main>` horizontal padding corrected from Phase 2 `px-8` to `px-4 sm:px-8` to prevent horizontal scroll at 375px. Source: 04-RESEARCH.md Pitfall 2.

## Typography

Extends Phase 2 contract (source: `02-UI-SPEC.md`). Phase 4 adds two new roles: Display (result amount) and Subtitle (header subtext).

| Role | Size | Weight | Line Height | Tailwind class |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (regular) | 1.5 | `text-base leading-normal` |
| Label / Section heading | 16–20px | 600 (semibold) | 1.2–1.5 | `text-base font-semibold` / `text-xl font-semibold leading-tight` |
| Secondary / Citation | 14px | 400 (regular) | 1.5 | `text-sm leading-normal` |
| Display (result amount in sticky bar) | 24px | 600 (semibold) | — (single line) | `text-2xl font-semibold` |

**Constraints:**
- Exactly 2 font weights: 400 (regular) and 600 (semibold). No bold (700) or black (900) introduced.
- Font sizes in use: 14px, 16px, 20px, 24px — 4 sizes, within the 3–4 size limit.
- Header subtitle: `text-sm text-blue-200` (14px regular, blue-200 for contrast on blue-800 bg).
- Sticky bar zero-state message: `text-base text-blue-100` — body size, lighter tone, not semibold.
- DRAFT overlay text: `text-[12vw] font-semibold uppercase tracking-widest` — viewport-relative, decorative watermark. Uses semibold (600) consistent with the 2-weight contract; the stamp aesthetic is achieved via `rotate-[-30deg] opacity-15 uppercase tracking-widest`, not via weight.

## Color

Extends and deepens Phase 2 contract (source: `02-UI-SPEC.md`, `04-CONTEXT.md Decisions`).

| Role | Value | Tailwind class | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | #ffffff / #f9fafb | `bg-white` / `bg-gray-50` | Page background; fieldset backgrounds |
| Secondary (30%) | #1e40af (blue-800) | `bg-blue-800` | Page header bar; sticky result bar |
| Accent (10%) | #1d4ed8 (blue-700) | `text-blue-700` / `ring-blue-700` | Focus rings on form controls; hyperlink text (footer links); `accent-blue-600` on radio/checkbox checked state |
| Destructive | not applicable | — | No destructive actions in Phase 4 |
| DRAFT overlay | #dc2626 (red-600) | `text-red-600 opacity-15` | DRAFT watermark only |

**Accent reserved for:** Focus rings on interactive form controls (`<input type="radio">`, `<input type="checkbox">`, `<select>`) and hyperlink text (footer external links). NOT used for headings, body text, borders, or decorative elements. Source: 04-CONTEXT.md Decisions (civic visual design).

**Text color assignments (full page):**
- Header bar + sticky bar: `text-white`
- Header subtitle: `text-blue-200`
- Sticky bar secondary (breakdown line): `text-blue-100` or `text-white/80`
- Page heading `<h1>`: removed from `<main>` (moved into header bar) — inherits `text-white`
- Primary content text: `text-gray-900` (inherited from Phase 2)
- Description text: `text-gray-600` (inherited from Phase 2)
- Citations: `text-gray-500` (inherited from Phase 2)
- Fieldset borders: `border-gray-200` (inherited from Phase 2)
- Breakdown tooltip background: `bg-blue-900 text-sm text-white`

**Note on blue-700 vs blue-800:** CONTEXT.md specifies blue-700/blue-800 for the bars. Use `bg-blue-800` for the header bar and sticky result bar (dominant secondary surface), and `bg-blue-900` for the breakdown tooltip (one step darker to distinguish). `blue-700` (`text-blue-700`) is reserved for accent use (focus rings and footer hyperlinks). Source: 04-CONTEXT.md Decisions.

## Layout Contract

Extends Phase 2 single-column layout. Phase 4 adds a full-width header above `<main>` and a footer below, plus the fixed sticky bar.

**Page document order:**
1. `<header>` — full-width blue bar (site name + subtitle)
2. `<main>` — centered `max-w-2xl`, `pb-16` bottom padding
   - Staffing Level fieldset (unchanged)
   - Collections Budget fieldset (unchanged)
   - Participating Cities fieldset (unchanged)
   - _(no result div — removed)_
3. `<footer>` — full-width gray bar (attribution + civic links)
4. DRAFT overlay `<div>` (if `config.draft === true`) — `fixed inset-0 z-40`
5. Sticky result bar `<div id="result-bar">` — `fixed bottom-0 z-50`
6. `<script>` blocks (unchanged load order; `url.js` added after `calculator.js`)

**Sticky bar `<div id="result-bar">` is `relative` so the breakdown tooltip can use `absolute bottom-full`.**

**z-index hierarchy:**
- Content: default (auto)
- DRAFT overlay: `z-40`
- Sticky result bar: `z-50`
- Breakdown tooltip: within `z-50` stacking context (no explicit z needed)

**Mobile viewport (375px) constraints:**
- Header: `px-4` → 16px padding each side, 343px content width. Safe.
- `<main>`: `px-4 sm:px-8` → 343px content width on mobile. Safe.
- Sticky bar: `left-0 right-0` → full viewport width. Safe.
- No horizontal scroll possible with these constraints.

## Interaction Contract

| Interaction | Behavior | Implementation |
|-------------|----------|----------------|
| Form control change | URL updated via `history.replaceState`; result bar updates | `calculator.js` targets `#result-bar`; `url.js` calls `replaceState` on `form.change` |
| Page load with URL params | Form restored to encoded selections before first result calculation | `url.js` `DOMContentLoaded` handler reads `URLSearchParams`, sets form values, dispatches `change` event |
| Invalid/stale URL params | Silently ignored; fallback to defaults | Unknown staffing ID → use first option; unknown city → skip checkbox; unknown collections value → use default. No error banner. |
| Breakdown toggle tap/click | `#breakdown-detail` toggles `hidden` attribute | Button click handler toggles `hidden` on tooltip |
| Click outside breakdown | `#breakdown-detail` hides | `document.addEventListener('click')` checks `contains`; hides if outside both tooltip and toggle button |
| Zero cities selected | Sticky bar shows message instead of dollar amount | `calculator.js` writes "Select at least one city" to `#result-amount` when city count is 0 |
| DRAFT overlay | Click-through — form fully interactive beneath it | `pointer-events: none` on overlay element |
| Keyboard: breakdown toggle | Operable via Enter/Space | Native `<button>` element handles this automatically |
| Focus state on breakdown toggle | Visible white outline | `focus-visible:outline focus-visible:outline-2 focus-visible:outline-white focus-visible:outline-offset-2` |

**URL encoding format:**
- Staffing: `?staffing={level.id}` — single value, matches existing radio `value` attributes
- Collections: `&collections={opt.value}` — single value, matches `<select>` `option.value`
- Cities: `&cities={id1},{id2}` — comma-joined city IDs (comma percent-encodes to `%2C`; browsers display decoded in address bar; no ambiguity since city IDs are simple slugs)
- Read back: `params.get('cities').split(',')` filter against `window.LIBRARY_DATA.cities` IDs
- Update mechanism: `history.replaceState(null, '', '?' + params.toString())` — no history stack entries

**JS file pattern:** Both `calculator.js` (updated) and `url.js` (new) follow IIFE + `'use strict'` module encapsulation. Source: 04-CONTEXT.md Established Patterns.

## Accessibility Notes

Full WCAG 2.1 AA compliance established in Phase 3 (TRST-03). Phase 4 must not introduce violations.

- **Sticky result bar `aria-live`:** `aria-live="polite" aria-atomic="true"` on `<div id="result-bar">` — replaces the Phase 3 `aria-live` region that was on `#result`. Source: 04-CONTEXT.md Result placement decisions.
- **Breakdown toggle:** Native `<button>` element — keyboard-operable (Enter/Space) by default. `aria-label` updated on expand/collapse.
- **DRAFT overlay:** `aria-hidden="true"` — purely decorative; screen readers skip it.
- **Header `<h1>`:** Moved from `<main>` to `<header>` — document outline still correct; `<h1>` appears once per page.
- **Footer external links:** `<a href>` with descriptive text (from `footerLinks[].label`) — no "click here" patterns.
- **Color contrast:** White text on blue-800 background — blue-800 is `#1e40af`, white is `#ffffff`. Contrast ratio ~8.6:1. Passes WCAG AA (4.5:1 for normal text, 3:1 for large text).
- **DRAFT overlay contrast:** Red-600 at 15% opacity on white — purely decorative; `aria-hidden="true"` exempts it from contrast requirements.
- **Focus ring on breakdown toggle:** `focus-visible:outline-white` on blue-800 background — white outline on blue is visually distinct and passes contrast.
