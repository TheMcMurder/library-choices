## Design System

| Property | Value |
|----------|-------|
| Tool | none — Tailwind CSS v4 utility classes only |
| Preset | not applicable |
| Component library | none — native HTML form elements only |
| Icon library | none — no icons in Phase 2 scope |
| Font | system-ui stack (Tailwind v4 default) |

**Rationale:** The project is Eleventy + Tailwind CSS v4 + vanilla JS — not a React/Vite/Next.js project. shadcn is not applicable. Phase 2 renders only native `<input>`, `<label>`, `<fieldset>`, `<legend>`, and `<cite>` elements. No custom component library is needed.

**Note:** Tailwind v4 has no `tailwind.config.*` file. All customization is done via CSS `@theme` blocks in `src/css/style.css` if needed. The current file is bare (`@import "tailwindcss"`). This phase will introduce the first utility token declarations if any are needed beyond the default scale.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gap between checkbox/radio and its label |
| sm | 8px | Gap between label and description text within a control |
| md | 16px | Gap between individual controls within a fieldset; default `<p>` margin |
| lg | 24px | Internal fieldset padding; gap between fieldset sections |
| xl | 32px | Gap between the three major form sections (Staffing, Collections, Cities) |
| 2xl | 48px | Outer page padding (vertical); space above the result placeholder |
| 3xl | 64px | Not used in Phase 2 |

**Tailwind v4 utility mapping:**

| Token | Tailwind class |
|-------|---------------|
| xs | `gap-1` / `space-x-1` |
| sm | `gap-2` / `mt-2` |
| md | `gap-4` / `mt-4` / `space-y-4` |
| lg | `gap-6` / `p-6` |
| xl | `gap-8` / `mt-8` |
| 2xl | `py-12` / `px-8` |

Exceptions: none. Touch target minimum (44px) is not enforced in Phase 2 — that is Phase 3 TRST-03 scope. Controls in Phase 2 use default browser sizing; Phase 3 will add `min-h-[44px] min-w-[44px]` constraints.

## Typography

Source: `index.html` scaffold already uses `text-2xl` for `<h1>` and `text-gray-600` for body-secondary. These are honored as the established baseline. The `<h1>` weight is updated to `font-semibold` (600) to conform to the 2-weight rule.

| Role | Size | Weight | Line Height | Tailwind class |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (regular) | 1.5 | `text-base leading-normal` |
| Label | 16px | 600 (semibold) | 1.5 | `text-base font-semibold leading-normal` |
| Description | 14px | 400 (regular) | 1.5 | `text-sm leading-normal` |
| Heading / Legend | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight` |

**Constraints:**
- Exactly 2 font weights used: 400 (regular) and 600 (semibold).
- Exactly 3 font sizes used: 14px (description/cite), 16px (body/label), 20px (section headings and fieldset legends).
- `<cite>` elements use 14px regular, color `text-gray-500`, no italics override needed (browser default italic is acceptable for citations).
- Page `<h1>` uses `text-2xl font-semibold` (24px/600). The Phase 1 scaffold used `font-bold` (700); this is corrected to `font-semibold` (600) to maintain exactly 2 font weights across the page.

## Color

Source: Phase 1 scaffold established `bg-white text-gray-900` as the surface. These are locked.

| Role | Value | Tailwind class | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | #ffffff | `bg-white` | Page background, form surface |
| Secondary (30%) | #f9fafb | `bg-gray-50` | Fieldset backgrounds — visually group each control section |
| Accent (10%) | #2563eb | `text-blue-600` / `ring-blue-600` | Focus rings on inputs only; checked radio/checkbox indicator color (browser default) |
| Destructive | not applicable | — | No destructive actions in Phase 2 |

**Accent reserved for:** Focus rings on interactive form controls (`<input type="radio">`, `<input type="checkbox">`) only. The accent color is NOT used for headings, body text, borders, or any decorative purpose in Phase 2.

**Text color assignments:**
- Primary text (headings, labels): `text-gray-900`
- Body/description text: `text-gray-600`
- Citation/secondary metadata: `text-gray-500`
- Fieldset borders: `border-gray-200`

## Layout Contract

- **Single-column, mobile-first.** All content in one column. No sidebar. No grid layout in Phase 2.
- **Max width:** `max-w-2xl` (672px) centered. Comfortable reading width for form controls with explanatory copy.
- **Section order:** Page heading → Staffing Level fieldset → Collections Budget fieldset → Participating Cities fieldset → Result placeholder.
- **Fieldset spacing:** `space-y-8` between the three fieldsets (32px gap, mapped to `xl` token).
- **Control spacing within fieldset:** `space-y-4` between individual radio/checkbox controls (16px gap, mapped to `md` token).

## Interaction Contract

Phase 2 has no JavaScript interactions. All interactivity is native browser form behavior.

| Interaction | Behavior | Notes |
|-------------|----------|-------|
| Radio button selection | Native browser — single selection within `name="staffing"` group | First option pre-checked via `{% if loop.first %}checked{% endif %}` |
| Collections checkbox | Native browser toggle — checked by default | No JS handler in Phase 2 |
| City checkbox | Native browser toggle — all checked by default | `data-households` attribute set for Phase 3 |
| Focus state | Browser default focus ring, overridden with `accent-blue-600` for checked state | Phase 3 will verify keyboard nav |
| Form submission | None — no `<form action>` or submit button in Phase 2 | Form exists as semantic grouping only |

## Accessibility Notes (Phase 2 Baseline)

Full WCAG 2.1 AA compliance is Phase 3 scope (TRST-03). Phase 2 must not introduce violations.

- Every `<input>` has a corresponding `<label for="...">` with matching `id`.
- Every group of related inputs is wrapped in `<fieldset>` with `<legend>`.
- `<cite>` is used semantically for source citations (not purely for italics).
- No color-only information conveyance in Phase 2.
- `lang="en"` on `<html>` is already present from Phase 1.
