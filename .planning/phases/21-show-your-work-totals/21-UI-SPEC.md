## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, popover padding (`px-4 py-2` = 16px / 8px) |
| md | 16px | Default element spacing, popover horizontal padding |
| lg | 24px | Section padding (fieldset `p-6` = 24px) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Result bar uses `py-4` (16px vertical) and `px-4` (16px horizontal) — no exception needed.
- Popover `#breakdown-detail` uses `px-4 py-2` (16px / 8px) — formula rows add `py-1` (4px) row padding.
- Touch target minimum: 44px height enforced on `#breakdown-toggle` via `p-2` button padding around 20px icon = ~36px tap area; this is the existing pattern and is not changed in this phase.

Source: `src/index.html` class inspection.

## Typography

All text is rendered in the browser system font stack. Tailwind base font size is 16px.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / formula row text | 14px (`text-sm`) | 400 (normal) | 1.5 |
| Secondary bar line (total cost) | 14px (`text-sm`) | 400 (normal) | 1.25 |
| Primary bar amount | 24px (`text-2xl`) | 600 (semibold) | 1.2 |
| Total row in formula | 14px (`text-sm`) | 600 (semibold) | 1.5 |

Notes:
- The popover `#breakdown-detail` already uses `text-sm` (14px). All formula content stays within `text-sm`. Source: `src/index.html` line 160.
- The primary result bar display (`#result-amount`) uses `text-2xl font-semibold` for the per-household figure. The new secondary total line uses `text-sm` with subdued color to establish visual hierarchy. Source: `src/js/calculator.js` line 55.
- The Total row in the formula uses `font-semibold` to match accounting convention (sum is visually distinct from addends).

## Layout Contract — Breakdown Popover Formula

Layout approach: `<table>` with two columns — amounts (right-aligned) and labels (left-aligned). This satisfies D-05's accounting-style alignment reliably on mobile at 375px without relying on flex gap approximations.

```
#breakdown-detail (bg-blue-900, text-white, text-sm, px-4 py-2)
  <table> (w-full)
    <tbody>
      <tr> [Hours Open row]
        <td class="text-right pr-3 tabular-nums">$56,440</td>
        <td class="text-blue-200">Hours Open</td>
      </tr>
      <tr> [Digital row]
        <td class="text-right pr-3 tabular-nums">$55,000</td>
        <td class="text-blue-200">Digital</td>
      </tr>
      <tr> [Physical row]
        <td class="text-right pr-3 tabular-nums">$15,000</td>
        <td class="text-blue-200">Physical</td>
      </tr>
    </tbody>
    <tfoot>
      <tr> [Separator — full-width hr via colspan]
        <td colspan="2" class="pt-1 pb-1">
          <hr class="border-blue-700" />
        </td>
      </tr>
      <tr> [Total row]
        <td class="text-right pr-3 tabular-nums font-semibold">$126,440</td>
        <td class="font-semibold">Total</td>
      </tr>
      <tr> [Division equation]
        <td colspan="2" class="pt-1 text-blue-200">
          $126,440 ÷ 1,234 households = $102.46/year
        </td>
      </tr>
    </tfoot>
  </table>
```

Source: CONTEXT.md D-05, D-06; codebase pattern for `<table>` already used inside staffing level cards (`src/index.html` lines 49-64).

## Collapsed Bar Layout Contract

`#result-amount` changes from a single text node to two lines:

```
<span id="result-amount">
  <span class="block text-2xl font-semibold">$102.46/household/year</span>
  <span class="block text-sm text-blue-200">$126,440 total</span>
</span>
```

The `block` display on each inner span stacks the two lines vertically. The outer `#result-amount` retains its existing class assignment from JS (`text-2xl font-semibold`) — the JS must switch to `innerHTML` instead of `textContent` for this element.

Source: CONTEXT.md D-01; `src/js/calculator.js` lines 45-46 and 55-56.

## Interaction Contract

| State | Trigger | Behavior |
|-------|---------|----------|
| Collapsed (default) | Page load / any form change | `#result-amount` shows two lines: primary per-household + secondary total |
| Expanded popover | Click `#breakdown-toggle` (info icon) | `#breakdown-detail` `hidden` removed; formula table shown above bar |
| Popover dismiss | Click outside popover + toggle btn | `#breakdown-detail` `hidden` set; toggle aria-label resets to "Show cost breakdown" |
| No cities selected | Any city unchecked to zero | `#result-amount` single line: "Select at least one city" in `text-base text-blue-100`; `#breakdown-detail` content cleared |
| Screen reader announcement | Any form `change` / `input` event | `aria-live="polite" aria-atomic="true"` on `#result-bar` announces full bar content — formula must read naturally when spoken |

Accessibility requirement: the division equation line must be readable as a sentence when announced by a screen reader. Use: `$126,440 ÷ 1,234 households = $102.46 per year` (spell out "per year" for SR, or use `aria-label` on the element to override the `÷` symbol if testing reveals pronunciation issues).

Source: CONTEXT.md code_context integration points; `src/index.html` line 153 `aria-live` attribute; `src/js/calculator.js` lines 42-47.
