## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-only gaps (`gap-1`), inline tick button padding (`px-1 py-1`) |
| sm | 8px | Compact internal spacing (`space-y-2`, `mt-1`) |
| md | 16px | Default element spacing — flex gap between sliders (`gap-4`), form section spacing (`space-y-4`) |
| lg | 24px | Fieldset internal padding (`p-6`) |
| xl | 32px | Between major form sections (`space-y-8`) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets: `min-h-[44px]` on interactive cards (existing pattern, Phase 10). Slider tick buttons use `px-1 py-1` (8px vertical, 4px horizontal) — acceptable for non-primary touch targets.
- Slider thumb: 20px diameter (`h-5 w-5` via `style.css`) — not on the 8-point scale but governed by the range input CSS already in the project.
- Divider between the two sliders inside the fieldset: `<hr class="border-gray-200" />` — no spacing token, consistent with existing visual separator pattern.

Source: Direct inspection of `src/index.html` and `src/css/style.css`. No changes to spacing scale in this phase — the new divider between the two sliders follows the existing fieldset pattern.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / description | 14px (`text-sm`) | 400 regular (`font-normal`) | 1.5 (`leading-normal`) |
| Label / slider label | 16px (`text-base`) | 600 semibold (`font-semibold`) | 1.5 (inherits) |
| Fieldset legend | 20px (`text-xl`) | 600 semibold (`font-semibold`) | 1.2 (inherits) |
| Result bar amount | 24px (`text-2xl`) | 600 semibold (`font-semibold`) | 1.25 (inherits) |

Additional type elements in this phase:
- Slider tick labels: 14px (`text-sm`), weight toggles between 400 (`font-normal`, inactive) and 600 (`font-semibold`, active or current-level), no explicit line-height set.
- Source citations: 14px (`text-sm`), weight 400, `not-italic` override (existing pattern).

Source: Direct inspection of `src/index.html` — these sizes and weights are already established and must be matched exactly by the new macro output.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-white` / `text-gray-900` | Page background (`body`), main content text |
| Secondary (30%) | `bg-gray-50` / `border-gray-200` | Fieldset backgrounds, card backgrounds, footer |
| Accent — interactive (10%) | `blue-800` / `blue-600` | Slider track thumb (`blue-800`), active tick label (`text-blue-800`), slider track (`blue-200`), selection ring (`ring-blue-600`), focus outline (`outline-blue-600`), result bar background (`bg-blue-800`) |
| Accent — current level | `amber-500` / `amber-600` | Current-level card ring (`ring-amber-500`), current-level badge background (`bg-amber-500`), current-level tick label text (`text-amber-600`) — amber-600 used for tick (not amber-500) for WCAG AA contrast on white |
| Destructive | Not applicable | No destructive actions in this phase |

Accent reserved for:
1. `blue-800`: slider track thumb, active tick label text (`text-blue-800`), result bar background
2. `blue-600`: selection ring on checked cards, focus outline on interactive elements
3. `blue-200`: slider track background
4. `amber-500`: current-level card border ring, current-level badge background
5. `amber-600`: current-level tick button text (contrast-safe on white)

Primary focal point: result bar (`bg-blue-800`, `text-2xl` amount figure) — highest contrast element on the page.

Phase 14 does not introduce any new colors. Both new sliders use the same blue/amber/gray pattern as the existing slider. The `data-current-level="true"` + JS guard preserves `text-amber-600` on current-level ticks across both new sliders.

Source: CONTEXT.md (D-09), STATE.md (Phase 13 decisions: amber-600 for tick, ring-amber-500 for card), direct inspection of `src/index.html` and `src/css/style.css`.

## Component Inventory

New components introduced in this phase:

### 1. Nunjucks Slider Macro (`src/_includes/macros/slider.njk`)

Macro name: `collectionSlider`

Parameters:
| Param | Type | Description |
|-------|------|-------------|
| `id` | string | Element id for the range input (e.g. `collections-digital`) |
| `name` | string | Form input name attribute |
| `label` | string | Human-readable label (e.g. `Digital collections budget`) |
| `options` | array | Array of `{value, description, isCurrentServiceLevel?, isDefault?}` |
| `source` | string | Source citation text |
| `amountId` | string | Id for the dollar amount display span |
| `descriptionId` | string | Id for the description paragraph |

Rendered structure per slider:
```
div.flex.flex-col.gap-4
  div.flex.items-center.justify-between
    label[for={id}] .text-base.font-semibold.text-gray-900
    span#{amountId} .text-base.font-semibold.text-blue-800
  input[type=range]#{id} .w-full.cursor-pointer
  div.flex.justify-between.px-0[aria-hidden=true]
    button[data-slider={id}][data-value={opt.value}][data-current-level=true (if isCurrentServiceLevel)]
      .text-sm.px-1.py-1 — inactive: .text-gray-500.font-normal
                         — active:   .text-blue-800.font-semibold
                         — current:  .text-amber-600.font-semibold (preserved by JS guard)
  p#{descriptionId} .text-sm.text-gray-600.leading-normal
  cite .text-sm.text-gray-500.not-italic
```

## Interaction Contract

### Slider Behavior (both sliders, identical pattern)

| Interaction | Behavior |
|-------------|----------|
| Drag range input | Updates amount span and description paragraph in real time via `input` event |
| Click tick button | Sets range input value to tick's `data-value`, dispatches `input` + `change` events |
| Hover tick button | `hover:text-blue-600` |
| Focus tick button | `focus-visible:outline focus-visible:outline-2 focus-visible:outline-blue-600 focus-visible:outline-offset-2` |
| Active tick styling | `text-blue-800 font-semibold` |
| Inactive tick styling | `text-gray-500 font-normal` |
| Current-level tick styling | `text-amber-600 font-semibold` — preserved by JS guard regardless of active/inactive state |
| Page load from shared URL | `delta` param restores digital slider; `tau` param restores physical slider; both `updateSliderLabels()` calls run on init |

### Scoping Rule (critical — prevents cross-slider interference)

Each tick button carries `data-slider="{id}"`. All JS queries that update tick styling must be scoped to `[data-slider="{id}"]`, not `[data-value]` globally.

### Accessibility

- Range inputs have `aria-valuetext` set by JS: `"{amount} dollars — {description}"`
- Tick button row has `aria-hidden="true"` — navigated via range input only by keyboard/SR
- Focus outline on range input: `outline: 2px solid blue-600; outline-offset: 2px` (via `style.css`)
- Both sliders are inside a `<fieldset>` with `<legend>Collections Budget</legend>` — SR announces fieldset context for both

Source: RESEARCH.md (Pattern 1, Pattern 4), CONTEXT.md (D-08, D-09), STATE.md (Phase 12 focus ring decisions, Phase 10 sr-only pattern).
