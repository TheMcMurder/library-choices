## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element gap (`gap-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `mt-2`) |
| md | 16px | Default element spacing (`px-3 py-2` on select, `gap-4`) |
| lg | 24px | Fieldset internal padding (`p-6` = 24px), section spacing (`space-y-4`) |
| xl | 32px | Page horizontal padding (`px-8`) |
| 2xl | 48px | Page vertical padding (`py-12` = 48px) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch target minimum: 44px (WCAG 2.1 SC 2.5.5). Radio and checkbox wrappers use `min-h-[44px]` — this is a non-scale value required by the accessibility contract. Source: REQUIREMENTS.md TRST-03 + RESEARCH.md Pattern 5.
- Result region top margin: `mt-8` (32px) — uses xl token.
- Form section gap: `space-y-8` (32px) — uses xl token.

## Interaction Contract

### Real-Time Calculation (CONF-04)

- Trigger: `change` event on `#configurator` via a single delegated listener
- Reads: staffing cost from `input[name="staffing"]:checked` `data-cost`, collections cost from `select#collections.value` (live property — not `data-cost`), households from `input[name="cities"]:checked` `data-households` (summed)
- Writes: innerHTML of `#result` (entire region replaced on each update)
- Initial call: `updateResult()` runs immediately at script load (script is at end of `<body>`, DOM is ready — no `DOMContentLoaded` needed)

### Zero-City Guard (CONF-05)

- Condition: `totalHouseholds === 0` before division
- Behavior: Write empty-state copy to `#result` and return — never divide
- Result: No `Infinity`, no `NaN`, no error value visible

### Accessibility (TRST-03)

| Requirement | Implementation |
|-------------|---------------|
| Screen reader announcement | `aria-live="polite"` + `aria-atomic="true"` on `#result` div |
| Keyboard navigation | Native `<input type="radio">`, `<input type="checkbox">`, `<select>` — operable by default; no JS needed |
| Visible focus indicator (radio/checkbox) | Add `focus-visible:outline focus-visible:outline-2 focus-visible:outline-blue-600 focus-visible:outline-offset-2` to each `<input type="radio">` and `<input type="checkbox">` |
| Visible focus indicator (select) | Already present: `focus:outline-none focus:ring-2 focus:ring-blue-600` |
| Touch target minimum 44px (radio/checkbox) | Change wrapper class from `flex items-baseline gap-2` to `flex items-center gap-2 min-h-[44px]` |
| Touch target minimum 44px (select) | `py-2 px-3` already provides adequate height; verify with DevTools and add `min-h-[44px]` if needed |

Source: REQUIREMENTS.md TRST-03 + RESEARCH.md Patterns 4, 5, 6
