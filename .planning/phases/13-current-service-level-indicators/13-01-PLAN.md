  <action>
Modify the `updateSliderLabels()` function in src/js/calculator.js. Replace lines 58-65 (the nodeButtons.forEach block) with a version that guards the current-level tick from being overwritten with gray classes.

Replace this existing block:
```js
var nodeButtons = document.querySelectorAll('[data-value]');
nodeButtons.forEach(function (btn) {
  var isActive = btn.dataset.value === String(slider.value);
  btn.classList.toggle('text-blue-800', isActive);
  btn.classList.toggle('font-semibold', isActive);
  btn.classList.toggle('text-gray-500', !isActive);
  btn.classList.toggle('font-normal', !isActive);
});
```

With this guarded version:
```js
var nodeButtons = document.querySelectorAll('[data-value]');
nodeButtons.forEach(function (btn) {
  var isActive = btn.dataset.value === String(slider.value);
  var isCurrentLevel = btn.dataset.currentLevel === 'true';
  btn.classList.toggle('text-blue-800', isActive);
  btn.classList.toggle('font-semibold', isActive || isCurrentLevel);
  btn.classList.toggle('text-gray-500', !isActive && !isCurrentLevel);
  btn.classList.toggle('font-normal', !isActive && !isCurrentLevel);
  btn.classList.toggle('text-amber-600', !isActive && isCurrentLevel);
});
```

Logic:
- `text-blue-800`: active tick (including current-level when it is the active one)
- `font-semibold`: active tick OR current-level tick (current-level always bold)
- `text-gray-500`: inactive tick that is NOT the current-level tick
- `font-normal`: inactive tick that is NOT the current-level tick
- `text-amber-600`: current-level tick when it is NOT the active tick

When the slider is at $30k (the current-level value), the tick shows blue-800 like any active tick. When the slider moves away, the tick shows amber-600 instead of gray-500.

Do NOT add any hardcoded values like "30000" in the JS -- the guard uses only the `data-current-level` attribute set by the template.
